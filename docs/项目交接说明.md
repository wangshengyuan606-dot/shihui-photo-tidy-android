## 1. 项目目标

这是一个“智能相册整理”Android App，目标是帮助用户浏览、筛选、保留、删除、压缩、归档照片和视频。

核心体验方向：

- 首页偏轻量、情绪化，主视觉是“照片盲盒”。
- 整理页按日期、相册、相似照片三类组织照片。
- 大图整理页支持滑动保留/删除、缩略图切换、标签归档、手势设置。
- 回收站、压缩页、我的页、设置项都使用同一套浅色纸纹背景和透明玻璃风格。

开源代码策略：

- 可以复用 MIT 许可项目的代码或思路。
- GPL 项目只看交互和产品思路，不搬源码。
- 如果后续上传 GitHub，需要在 README 和 THIRD_PARTY_NOTICES 中说明参考来源和许可。

## 4. 当前 UI 状态

### 首页

主要文件：`HomeScreen.kt`

当前结构：

- 纸纹背景。
- 顶部“欢迎回来！”。
- 右上角是消息按钮，不是回收站。
- 搜索框可按关键词搜索照片。
- 搜索框已升级为高级搜索索引，可匹配文件名、相册名、用户标签、日期、照片/视频/截图类型、地点字段、人物命名字段。
- 搜索框已接入 ML Kit 图片标签，后台会为照片生成风景、宠物、猫、狗、食物、天空、建筑等内容关键词，支持搜索“风景”“宠物”“猫”“狗”“人物”“合照”等。
- 搜索框已接入 ML Kit 人脸检测，有人脸的照片会加入“人物/人脸/人像/自拍/合照”等搜索词。
- 搜索索引已持久化到本地 `SearchIndexStore`。应用重启后会先读取缓存，只有新照片或内容变更的照片才会重新补充索引。
- 地点索引会记录“已检查但无 GPS”的照片，避免没有地理信息的照片每次启动都被重复扫描。
- 地点字段由 `PhotoLocationExtractor` 在后台读取 EXIF GPS 并尝试反向地理编码生成。
- 人物命名字段已有 `PersonCluster` 数据结构和搜索接入；当前版本已能检测“有人脸”，下一步如果要区分同一个人并命名，需要接 FaceNet/MobileFaceNet 向量模型做人物聚类。
- 搜索框下方是相册标签，点击标签跳转对应相册。
- “照片盲盒”显示中间大图，两侧露出上一张/下一张照片。
- 盲盒卡片上有 `Add+`、保留、删除。
- 右侧有刷新按钮，点击重置本次时光盲盒照片。
- 底部是“照片概览”和自定义 Tab 栏。

### 相册页和相似页

主要文件：`TidyScreen.kt`

已实现方向：

- 整理页有日期、相册、相似三种模式。
- 相册模式参考中间大图相册，其他相册变扁变小。
- 相似照片由后台 `SimilarPhotoDetector` 自动识别，不再用临时假分组。
- 相似照片横排展示，相似组内最佳照片右上角显示黄色星标。
- 用户点击相似照片行或某张缩略图后，会进入大图整理页；该大图整理页只包含当前这一组相似照片。
- 从相似照片进入的大图整理页在整理完当前组或手动返回后，会回到整理页的“相似”列表。
- 识别方式：本地生成照片 dHash，按汉明距离、同日期/同相册/连拍时间窗口聚类，再用分辨率、文件大小、是否截图等规则推荐最佳照片。
- 识别只在本机后台运行，不上传照片。

后续可继续优化：

- 相册上下滑到底部时，最后一个文件夹不应异常放大。
- 相册滑动动效要继续贴近用户给的 Uiverse 放大相邻项效果，但需要保持移动端顺滑。
